What is this?

A proof-of-concept for data-over-sound communication using low frequencies (40–200 Hz). One device transmits tones through its speaker; another detects them via microphone using FFT (Fast Fourier Transform) analysis built into the browser.

The goal: test whether devices can communicate acoustically through walls using frequency-shift keying (FSK) — switching between two tones to encode binary data.

FSK Frequency Pairs

Two frequencies encode binary: Freq 0 = "zero", Freq 1 = "one". The frequencies must land in different FFT bins to be distinguishable. The bin gap indicator shows whether your chosen pair has enough separation.

Preset pairs use prime numbers that avoid 50 Hz mains harmonics (which would interfere with detection in countries using 50 Hz AC power, including Iran).

53/59 Hz — lowest pair, hardest to hear, 2 bin gap (tight). 53/67 Hz — low, 4 bin gap (solid). 73/97 Hz — conservative, 8 bin gap (very reliable).

Transmission Modes

Tone (Freq 0/1) — continuous single frequency. Use to test detection and range for each frequency independently.

FSK Message — encodes text as binary using the two frequencies. Includes a 16-bit alternating preamble for sync, a length byte, message bytes, and an end marker.

Testing Tips

1. Open this page on two devices. One transmits, the other listens.

2. Start with Tone mode to confirm each frequency is detectable.

3. Check the bin analysis — if gap is 0, the pair won't work.

4. Try placing the transmitting device face-down on a hard surface to amplify bass.

5. Test at night when ambient noise is lowest.

6. Note: below ~50 Hz, some microphones may filter out the signal due to noise cancellation.
